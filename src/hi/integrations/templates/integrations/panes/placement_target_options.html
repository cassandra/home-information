{% comment %} Reusable dropdown options for placement target-selects (LocationViews + Collections). Renders LocationViews first as one per Location (ordered by Location.order_id, views within by LocationView.order_id), then a single 'Collections' . Option values are *tagged* — ``view:`` or ``collection:`` — so views and collections can share the same select element without colliding on numeric ids. Optionally honors ``selected_view_id`` / ``selected_collection_id`` (passed via ``include … with …``) by adding the ``selected`` attribute to the matching option. Used by the placement's top dropdown so the smart-default sticks; group/entity dropdowns don't pass them.
